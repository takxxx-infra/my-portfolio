## プロジェクト概要
- 顧客プライベートクラウドの更改案件を横断し、Citrix Netscaler のバージョンアップ、余剰サーバの仮想化再活用、メール通知基盤刷新、Zabbix 監視最適化を担当。
- プロジェクトマネージャーと並走し、要件整理〜設計〜構築〜運用手順化までを 1 名で回すタスクと、3〜6 名チームのテクニカルリードを兼務。

## 主な取り組み
1. **ロードバランサ移行**  
   - 旧構成の不要設定を洗い出し、命名規則や成果物格納ルールを再定義。  
   - Netscaler 12.1→13.1 への移行計画、要件整理、タスク分解、メンバーへの配賦を実施。
2. **仮想化基盤構築**  
   - 余剰サーバで Proxmox VE を採用し、用途別の仮想ネットワーク/運用ルールを設計。  
   - ポートフォワーディング簡略化スクリプトや運用マニュアルを整備し、検証〜本番を短期間で完了。
3. **メール通知基盤刷新**  
   - Rocky Linux + Postfix/Dovecot をベースに TLS/SASL/DMARC を実装し、暗号化と認証を強化。  
   - AWS SES 連携や Bash/Python での構築スクリプトを作成し、3 カ月で設計〜リリース。
4. **監視・自動化**  
   - Zabbix API を活用し、移行作業を 25 時間 → 2 時間に短縮。  
   - Symantec Endpoint Protection、Zabbix Agent 約 200 台のバージョンアップを自動化。  
   - 運用監視ではアラート削減と通知制御、障害復旧スクリプトを継続的に改善。

## 成果
- ロードバランサ/仮想化/メール/監視の各サブプロジェクトを重複期間で完遂。  
- 自動化により人的工数を大幅削減、監視体制の UX を高め、顧客のプライベートクラウド運用品質を底上げした。
